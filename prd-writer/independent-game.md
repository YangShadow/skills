# 独立游戏 章节模板

适用：独立部署运营的完整游戏（消除类、解谜类、休闲类等）需求文档。

## 章节结构

按以下顺序生成草稿。每章标注置信度（✅ / ⚠️ / ❌）。

---

### 0. 名词解释（按需）

> 仅在存在特殊领域词汇时生成此章。无则跳过。

格式：
| 名词 | 解释 |
|------|------|
| [术语] | [1-2 句解释，面向不了解业务的研发/设计] |

---

### 1. 需求背景

> 说明这款游戏为什么要做。

包含：
- 业务动机（拉新 / 促活 / 商业转化 / 节日营销等）
- 用户痛点或行为洞察
- 当前缺失的能力或机会点

---

### 2. 设计目标

> 这款游戏要实现什么效果。

按三类列举（有则写，无则略）：
- **核心体验**：用户玩起来感受到什么（爽感 / 挑战感 / 成就感）
- **商业转化**：如何引导道具购买 / 复活付费
- **留存钩子**：每日任务 / 关卡进度 / 排行榜竞争等

---

### 3. 核心玩法规则

> 完整描述机制，研发据此实现，设计据此出交互稿。

建议结构：
1. 【机制摘要】一段话概括整体玩法逻辑
2. 【关卡结构】
   - 关卡总数 / 阶段划分
   - 难度曲线（初期引导 / 中期核心 / 后期挑战）
   - 关卡解锁条件（通关 / 星级 / 道具消耗）
3. 【道具体系】
   - 道具类型（辅助类 / 消耗类 / 永久类）
   - 获取途径（关卡奖励 / 活动赠送 / 内购）
   - 消耗规则（每次使用数量 / 叠加效果）
4. 【复活机制】
   - 免费复活次数（每日上限）
   - 付费复活价格与规则
   - 复活后状态还原规则
5. 【存档策略】
   - 本地缓存 + 云端同步规则
   - 多设备冲突处理（以哪端为准）
   - 掉线期间进度保护

### 5. 交互与状态枚举

> 产品和设计对齐用。

建议结构：
1. **主要流程**（游戏入口 → 关卡选择 → 游戏进行中 → 通关/失败 → 结算）
2. **关键页面列表**（大厅 / 关卡地图 / 游戏主界面 / 结算页 / 道具商店等）
3. **状态枚举**：

| 状态 | 触发条件 | 展示内容 |
|------|---------|---------|
| 游戏进行中 | 进入关卡 | 游戏主界面 + 道具栏 + 剩余步数/时间 |
| 通关 | 达成目标 | 通关动效 + 星级评定 + 下一关按钮 |
| 失败 | 步数/时间耗尽 | 失败提示 + 复活选项 + 返回按钮 |
| 复活中 | 点击付费复活 | 支付确认弹窗 |
| 无网络 | 断网检测 | 断网提示 + 本地进度保护说明 |
