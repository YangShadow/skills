### 1. 需求背景

> 说明录音棚功能为什么要做。

包含：
- 业务动机（UGC 内容生产 / 用户创作留存 / 社交分享等）
- 用户痛点或行为洞察
- 当前缺失的能力或机会点

---

### 2. 设计目标

> 这个录音棚要实现什么效果。

按三类列举（有则写，无则略）：
- **核心体验**：用户录唱的流畅感 / 成就感
- **内容生态**：作品发布后的传播与互动
- **留存钩子**：收藏歌曲 / 历史作品 / 好友互听等

---

### 3. 核心玩法规则

> 完整描述录唱流程，研发据此实现，设计据此出交互稿。

建议结构：
1. 【流程摘要】一段话概括录唱完整链路
2. 【选歌】
   - 歌曲库来源与版权说明
   - 搜索 / 分类 / 推荐入口
   - 伴奏试听规则
3. 【调音】
   - 人声 / 伴奏音量调节范围（0–100）
   - 混响 / 音效预设选项
   - 音调调节（升降调范围）
4. 【录制】
   - 开始录制条件（倒计时提示）
   - 暂停 / 继续 / 重录规则
   - 实时波形展示
5. 【打断与恢复机制】
   - 来电打断：暂停录制，恢复后可续录或重录
   - 切后台：超过 N 秒自动暂停，返回后提示继续或放弃
   - 弱网 / 断网：本地录制不中断，上传阶段重试
6. 【预览】
   - 人声 + 伴奏合并后的预览播放
   - 调整音量后重新预览

---

### 3.5 作品发布

> 描述录唱完成后的发布流程与内容状态机。

包含：
1. 【状态机】

| 状态 | 触发条件 | 用户可操作 |
|------|---------|---------|
| 草稿 | 录制完成，未提交发布 | 继续编辑 / 提交发布 / 删除 |
| 待审 | 提交发布，等待审核 | 撤回 |
| 已发布 | 审核通过 | 分享 / 下架 |
| 已下架 | 用户主动下架 / 违规下架 | 重新发布（如合规） |

2. 【审核流程】
   - 触发时机（提交发布时）
   - 审核内容（歌词合规 / 音频内容检测）
   - 审核超时处理（超时自动通过 / 转人工）
3. 【版权检测】
   - 伴奏版权合规检测时机（选歌时 / 提交发布时）
   - 版权不合规的用户提示与处理

### 5. 交互与状态枚举

> 产品和设计对齐用。

建议结构：
1. **主要流程**（入口 → 选歌 → 调音 → 录制 → 预览 → 发布）
2. **关键页面列表**（歌曲库 / 录制主界面 / 调音面板 / 预览页 / 发布页等）
3. **状态枚举**：

| 状态 | 触发条件 | 展示内容 |
|------|---------|---------|
| 录制中 | 点击开始录制 | 波形动画 + 进度条 + 暂停按钮 |
| 暂停 | 点击暂停 / 来电 / 切后台 | 暂停状态 + 继续/重录按钮 |
| 预览中 | 录制完成，点击预览 | 播放波形 + 音量调节 + 重录按钮 |
| 上传中 | 点击发布 | 上传进度条 |
| 上传失败 | 网络异常 | 失败提示 + 重试按钮 |
| 审核中 | 上传成功 | 「作品审核中」状态提示 |
