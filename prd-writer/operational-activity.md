适用：直播 / 音乐业务的限时运营活动，含拉新、促活、节日营销等。

## 章节结构

按以下顺序生成草稿。每章标注置信度（✅ / ⚠️ / ❌）。

包含：
- 业务动机（拉新 / 促活 / 防流失 / 节点借势）
- 历史数据参考（如有：上次同类活动 ROI / 参与人数等）
- 本次活动的差异点或创新点

---

### 3. 目标用户

圈人规则，说明：
- 用户准入条件（等级范围、付费段位、历史行为标签）
- 排除条件（黑名单、已参与用户等）
- 预估覆盖人数

---

### 4. 活动规则

结构：
1. 参与条件（入口、前置行为）
2. 任务链路（用户需完成哪些动作，按步骤列出）
3. 奖励触发逻辑（满足何种条件 → 触发何种奖励）
4. 参与次数限制（每日 / 活动期间上限）
