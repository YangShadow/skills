# 语音歌房 章节模板

适用：语音歌房玩法模板（普通歌房、合唱房、多人游戏房等）需求文档。主要覆盖歌房基建，不涉及概率和 ROI 数值计算。

## 章节结构

按以下顺序生成草稿。每章标注置信度（✅ / ⚠️ / ❌）。

---

### 0. 名词解释（按需）

> 仅在存在特殊领域词汇时生成此章。无则跳过。

格式：
| 名词 | 解释 |
|------|------|
| [术语] | [1-2 句解释，面向不了解业务的研发/设计] |

---

### 1. 需求背景

> 说明这个歌房功能为什么要做。

包含：
- 业务动机（提升用户互动 / 增加留存 / 扩大社交场景等）
- 用户痛点或行为洞察
- 当前缺失的能力或机会点

---

### 2. 设计目标

> 这个歌房要实现什么效果。

按三类列举（有则写，无则略）：
- **核心体验**：用户在歌房里感受到什么（沉浸感 / 参与感 / 社交连接）
- **互动钩子**：上麦机制 / 合唱玩法 / 游戏互动等
- **运营目标**：DAU / 人均时长 / 麦位使用率等

---

### 3. 核心玩法规则

> 完整描述机制，研发据此实现，设计据此出交互稿。

建议结构：
1. 【机制摘要】一段话概括歌房整体逻辑
2. 【房间类型】
   - 普通歌房 / 合唱房 / 多人游戏房等类型说明
   - 各类型差异点
3. 【房间生命周期】
   - 创建 → 等待 → 进行中 → 结束 / 解散
   - 各状态转换条件
   - 房主下线后的处理逻辑（自动解散 / 转让房主）
4. 【麦位管理】
   - 麦位数量与布局
   - 上麦条件（自由上麦 / 申请上麦 / 邀请上麦）
   - 麦序规则（排队 / 抢麦）
   - 踢麦权限（房主 / 管理员）
   - 麦位状态（空麦 / 占麦 / 锁麦 / 禁麦）
5. 【角色与权限体系】

| 角色 | 权限 |
|------|------|
| 房主 | 创建房间、踢人、解散房间、设置管理员、锁麦/解麦 |
| 管理员 | 踢麦、禁麦、邀请上麦 |
| 麦上用户 | 发言、退麦 |
| 观众 | 申请上麦、收听 |

---

### 3.5 音频配置

> 音频相关技术参数，产品定义范围，研发据此做技术选型参考。

包含：
- **混流策略**：多路音频合并规则（主唱优先 / 均衡混流等）
- **回声消除**：是否开启 AEC（回声消除）/ ANS（噪声抑制）
- **合唱同步**：多端音频时序对齐方案（以服务端时间戳为准 / 客户端预缓冲等），允许的最大延迟阈值
- **音质参数**：采样率 / 码率要求

### 5. 交互与状态枚举

> 产品和设计对齐用。

建议结构：
1. **主要流程**（创建/加入房间 → 进入歌房 → 上麦/申请 → 演唱/互动 → 退出/解散）
2. **关键页面列表**（歌房主界面 / 麦位区 / 申请上麦弹窗 / 房间设置页等）
3. **状态枚举**：

| 状态 | 触发条件 | 展示内容 |
|------|---------|---------|
| 等待中 | 房间创建，人数不足 | 麦位空位提示 + 邀请入口 |
| 进行中 | 有人在麦上演唱 | 麦上用户头像 + 音频可视化 |
| 麦位申请中 | 用户点击申请上麦 | 申请排队状态 + 取消按钮 |
| 被踢下麦 | 房主/管理员操作 | 提示弹窗 + 回到观众席 |
| 房间解散 | 房主离开/主动解散 | 解散提示 + 返回首页 |
| 断线重连 | 网络恢复 | 恢复麦位状态 + 重新拉取房间信息 |
