## 流程

### 第一步：读取周报

读取当前目录下最新的周报文件（优先读取文件名含日期的 .md 文件，取日期最新的原始版本，不读整理版/微信版）。

### 第二步：确认输出目标

询问用户需要哪种格式：
- **Obsidian 阅读版**：结构化整理，含表格、标题层级
- **微信发送版**：纯文字，无表格，适合直接粘贴到微信
- **两者都要**

### 第三步：整理结构

按以下模块组织内容（根据实际周报内容灵活增减）：
1. 业务交付（按产品线分组，如秀场 / 语音 / 商城）
2. 游戏交付
3. 鹅票票交付
4. 技术优化（性能 / 鸿蒙 / 充值链路 / 减包专项 / AI / HTTPS反劫持等）

每个条目突出：项目名、当前状态、关键时间节点、重要数据。

### 第四步：生成对应格式

**Obsidian 版**
- 使用 Markdown 表格、多级标题、✅ 状态标注
- 关键数据加粗
- 保存为 `{原文件名}（整理版）.md`，存于同目录

**微信版**
- 禁用 Markdown 表格，改用缩进列表 + `·` 符号
- 用 `━━━━━━━━━━━━━━━━━━` 作为模块分隔线
- 用 `【模块名】` 作为分组标题
- 保留 `✅` 等简单符号，避免其他复杂 Markdown 语法
- 保存为 `{原文件名}（微信版）.md`，存于同目录
